# Информационная система «Автоматизация системы охраны труда»

Учебный дипломный проект на Django для учета сотрудников, инструктажей, программ обучения, средств индивидуальной защиты и проверок условий труда.

## Что умеет сайт

1. Хранит сотрудников, их подразделения, должности и контактные данные.
2. Ведет справочник программ обучения и инструктажей по охране труда.
3. Хранит перечень средств индивидуальной защиты.
4. Формирует нормы обеспечения СИЗ для отдельных программ и видов работ.
5. Регистрирует инструктажи сотрудников и их статус.
6. Рассчитывает плановую стоимость обеспечения СИЗ по нормам.
7. Ведет проверки условий труда и рекомендации по устранению нарушений.
8. Разделяет обычного пользователя и администратора.

Формула расчета:

```text
стоимость обеспечения = количество сотрудников x норма СИЗ x цена СИЗ
```

## Первый запуск на Windows

Откройте PowerShell в папке проекта и выполните команды по порядку:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
python manage.py migrate
python manage.py seed_data
python main.py
```

Открыть в браузере: `http://127.0.0.1:8000/`

Учебные пользователи:

| Роль | Логин | Пароль |
| --- | --- | --- |
| Администратор | `admin` | `admin123` |
| Пользователь | `student` | `student123` |

## Запуск через Docker

```powershell
docker compose up --build
```

Открыть в браузере: `http://127.0.0.1:8000/`

Контейнер сам выполняет миграции, создает учебные данные и собирает статические файлы. SQLite-база сохраняется в локальной папке `.docker-data`.

Для Render через Docker достаточно указать репозиторий с `Dockerfile`. Переменные окружения для деплоя:

```text
DEBUG=0
ALLOWED_HOSTS=.onrender.com
CSRF_TRUSTED_ORIGINS=https://*.onrender.com
SECRET_KEY=<любой длинный секретный ключ>
```

## Самые важные файлы

- `main.py` - запускает сайт.
- `core/models.py` - таблицы базы данных и расчет стоимости обеспечения СИЗ.
- `core/views.py` - логика страниц.
- `core/forms.py` - формы ввода.
- `core/urls.py` - адреса страниц.
- `templates/` - HTML-страницы.
- `static/css/app.css` - внешний вид.
- `core/management/commands/seed_data.py` - учебные данные.

## Связь с модулями дипломного проекта

- Модуль 1: модели в `core/models.py` соответствуют ER-диаграмме предметной области охраны труда.
- Модуль 2: Django создает SQL-таблицы через миграции.
- Модуль 3: метод `Order.material_cost()` рассчитывает стоимость обеспечения СИЗ.
- Модуль 4: формы, авторизация, роли и страницы сайта.
- Модуль 5: README, справка и описание структуры проекта.

Сейчас используется SQLite, чтобы проект запускался с нуля одной командой. Для итоговой версии можно подключить PostgreSQL без изменения пользовательской логики.
